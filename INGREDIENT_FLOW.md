# Gusto Expedition Admin UI (Ingredient) — Flow & Screens (Draft)

## 목표

* **어드민 UI는 Light/Neutral 업무용 대시보드 톤**을 따른다.
* 데이터 입력은 “재료 1개를 충분히 공부하고 다음으로 넘어가는” 방식이므로, **재료 상세 중심의 워크플로우**를 최적화한다.
* **다국어(i18n) 지원**을 전제로 한다.
* 관계는 기본적으로 **재료 상세 내에서 조작**하지만, **관계 전용 관리 페이지**도 별도로 필요(추후 범위).

---

## 1) 화면 구성 (Ingredient 기준)

### 1.1 재료 리스트 페이지 (Ingredient List)

**목적:** 재료 탐색/조회, 신규 재료 등록 진입

**요구사항**

* 재료 리스트 노출
* 상단에 다음 UI 요소 포함

  * 검색창
  * “재료 등록” 버튼
* 재료 항목 클릭 시 **재료 상세 페이지로 이동**

**추천(옵션)**

* 리스트 컬럼 예시: 이름(기본 언어), 태그, 생성일/수정일, 관계 수
* 필터: 언어, 태그, 생성일/수정일 정렬

---

### 1.2 재료 상세 페이지 (Ingredient Detail)

**목적:** 재료 내용 조회 + 수정/삭제 진입 + 관계 확인/조작(최종적으로는 3rd depth에서도 제공)

**요구사항**

* 상세 진입 시 기본은 **조회 모드(내용 표시 목적)**

  * “등록 화면과 동일한 포맷”으로 보여주되, input이 아니라 텍스트/뷰 형태로 표시
* 상단 또는 우측에 기능 제공

  * 수정(Edit)
  * 삭제(Delete)
* “수정” 클릭 시:

  * **등록과 동일한 포맷**으로 전환
  * 상세 페이지 내에서 **입력 가능한 영역들이 input으로 변경**되어 편집 가능 (View → Edit 토글)

**페이지 구조(등록 포맷과 동일)**

* 기본정보 섹션
* i18n 섹션 (다국어명, 설명, alias 등)
* 관계 섹션 (궁합/비궁합/중립 등)

---

## 2) 재료 등록 Flow (Multi-depth)

### 개요

* 리스트 페이지의 “재료 등록” 버튼 클릭 시 **여러 depth(step)로 등록 흐름 진행**
* 총 3 depth

---

### Depth 1 — 재료 기본정보 등록 (Ingredient Base)

**목적:** ingredient 테이블에 들어갈 기본 속성 입력

**입력 내용(예시)**

* 기본명(또는 기준 언어의 display name)
* (옵션) 태그/카테고리
* (옵션) 이미지/대표 아이콘
* (옵션) 공개 여부/상태값(draft/published 등)

**결과**

* Ingredient 기본 레코드 생성(또는 임시 저장) 후 다음 단계로 이동

---

### Depth 2 — 재료 i18n 정보 입력 (Ingredient i18n)

**목적:** ingredient_i18n(가칭) 테이블에 들어갈 다국어 데이터 입력

**요구사항**

* i18n 테이블에 들어갈 정보 입력
* i18n alias 입력(동의어/별칭) 입력

**입력 내용(예시)**

* 언어 선택 (ko / en / it ...)
* locale별 재료명
* locale별 설명(장문 가능)
* alias 목록(0..n)

  * 단일 입력 + 추가 버튼, 또는 태그 입력 UI 형태

**결과**

* 각 언어별 i18n 레코드 저장 후 다음 단계로 이동

---

### Depth 3 — 관계 조작 View (Ingredient Relations)

**목적:** 해당 재료의 관계(궁합/비궁합/중립 등)를 등록/수정

**요구사항**

* 관계를 조작할 수 있는 View 제공
* 관계 추가/편집은 “현재 재료(from)는 고정, 상대 재료(to)를 선택”하는 형태가 자연스러움

**기능(예시)**

* 탭: 궁합 / 비궁합 / 중립(또는 참고)
* 관계 추가:

  * to 재료 검색(자동완성)
  * 근거 메모(장문 가능)
  * (옵션) 강도/스코어
  * (옵션) 참고 링크
* 관계 편집/삭제:

  * 각 관계 항목 클릭 → 상세 편집(드로어/모달/인라인)

---

## 3) 상세 페이지와 등록 Flow의 관계

* 등록 Flow(Depth 1~3)에서 사용하는 **입력 포맷은 재료 상세 페이지의 Edit 모드와 동일**해야 한다.
* 재료 상세 페이지의 View 모드는 같은 레이아웃을 유지하되, 입력 컨트롤만 비활성/텍스트 표시로 변경한다.

---

## 4) 추후 고려(범위 밖이지만 방향성)

* 관계만을 조작/정리하기 위한 **관계 전용 페이지** 필요

  * from/to 필터, 타입 필터, 수정일 정렬, 중복 탐지 등
* 장문 메모 UX

  * 접기/펼치기
  * 전체화면 편집(에디터 확장)
  * 마크다운 지원 여부 검토
